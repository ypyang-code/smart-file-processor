## 2. 上传接口压测

### 测试目的

验证小文件上传接口的吞吐能力。

### 方法

参考 `docs/perf/test-upload.sh`（或 Windows Git Bash 等效脚本）：
```bash
# 循环上传 N 个文件，记录 HTTP 状态码和耗时
for i in $(seq 1 N); do
  curl -s -o /dev/null -w "%{http_code} %{time_total}\n" \
    -F "file=@testfile_${i}.txt" http://localhost:8080/api/file/upload
done
```

## 已知边界（Phase 5）

1. **PDF 解析非流式**：`PDFBox` 的 `writeText()` 仍会遍历全量页面对象树。文件大小（20MB）和页数（500 页）门禁配合控制内存峰值。
2. **无解析超时**：`Future.cancel()` 无法安全中断 PDFBox 底层 IO/解析线程。当前靠文件大小和页数门禁间接限制解析时间。
3. **解析失败状态不区分**：失败时 `file_info.status` 保持原有语义，错误原因仅记录在日志。后续可优化为 `error_reason` 字段。
4. **不包含 Outbox ES 同步延迟测试**：Phase 4 的 Outbox Scheduler 5 秒间隔导致的 ES 写入延迟为非实时设计的预期行为。
